Basics about IIS :

HTTP :
-> Hypertext transter protocol is a communications protocol
	that clients and servers use to communicate and exchange data
-> For data exchange to take place, Http depends on two other network protocols:
	TCP(Transmission Control Protocol) and IP(Internet Protocol).
-> HTTP is stateless, meaning the here is no like between two requests being
	successfully carried out on the same connection.


Anatomy of an HTTP Request :
- Establish a TCP connnection from the client to the server.
- Client initiates an HTTP request to the server.
- HTTP server response to the HTTP request.
- HTML page is loaded on client browser.

--> HTTP archive 

HTTP : User <-> Insecure COnnection <-> Normal HTTP
	3 way handshake using TCP, IP
HTTPS : User <-> Encrypted Connection <-> Secure HTTPS
	TLS(transport layer security) , SSH(Secure shell) handshake , Certificates
